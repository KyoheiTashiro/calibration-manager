# 5. 点検校正項目一覧(中核画面)

[画面設計 共通仕様・全体構成](./README.md) の一部。ステータス色・モーダル共通挙動・確認ダイアログ等の共通仕様は [README(§0 共通仕様)](./README.md#0-共通仕様) を参照。

**目的**: 全機器の点検校正項目を期限順で俯瞰し、フィルタと行アクションで日々の運用を回す中核画面。

**URL**: `/inspection-items`(クエリ `?status=` `?type=` `?execution=` `?personId=` を受け付ける)

## 表示項目

全機器の InspectionItem(稼働機器の有効項目)。列:

| 列         | 元                                                      | 備考                         |
| ---------- | ------------------------------------------------------- | ---------------------------- |
| ステータス | 導出([§0.3](./README.md#03-ステータスバッジ色共通定義)) | バッジ                       |
| 管理番号   | Equipment.managementNo                                  |                              |
| 機器名     | Equipment.name                                          |                              |
| 項目名     | InspectionItem.name                                     |                              |
| 種別       | type                                                    | 点検/校正                    |
| 内外       | execution                                               | 内部/外部                    |
| 周期       | cycle                                                   | 1M〜10Y                      |
| 担当       | Person.name(personId)                                   |                              |
| 最終実施日 | lastDoneDate                                            | 未実施は `—`                 |
| 次回期限   | nextDueDate                                             |                              |
| 発注推奨日 | 導出(§4.2)                                              | **外部のみ**表示、内部は `—` |

## 操作・アクション

- **フィルタ**: ステータス([§0.3](./README.md#03-ステータスバッジ色共通定義)の5値)/ 種別(inspection/calibration)/ 内外(internal/external)/ 担当(Person)。ダッシュボードからの遷移時は `status` クエリで初期化。
  - フィルタ状態の真実源は URL クエリパラメータ(`useSearchParams`)とし、ローカル state と二重管理しない。変更は `setSearchParams(..., { replace: true })` で反映し履歴を汚さない。不正・未知のクエリ値は無視して「全て」扱い(エラーにしない)。「クリア」は全クエリ除去(D-022)。
- 行アクション:
  - 「記録」→ **[実施記録登録モーダル](./07-record-modal.md)**。
  - 「案件」→ **[案件作成モーダル](./08-orders.md#案件作成モーダルplanned新規作成)**。**外部かつ有効な案件が無い項目のみ**表示。
  - 「編集」→ **[点検校正項目モーダル](./06-inspection-item-modal.md)**。

## 表示ルール・バリデーション

- 既定並び: **nextDueDate 昇順**。
- ステータスは [§0.3](./README.md#03-ステータスバッジ色共通定義) の単一導出関数で算出(保存値ではない)。フィルタ `status` は導出結果に対して適用。
- 対象は常に `status=active` 機器の `isActive=true` 項目のみ。「無効・非稼働も表示」トグルは不実装(D-023)。
- フィルタUIのみで自由入力なし。

## 空状態

- 「点検校正項目が未登録です」。フィルタ適用中は「条件に一致する項目はありません」+「クリア」。
