## 表示項目 / 機能

- **ご利用にあたって**: アプリの目的(点検・校正期限の一元管理)、データが LocalStorage に保存されサーバへ送信されないこと(オフラインでも利用可)、ブラウザ・端末ごとにデータが独立していること、定期バックアップの推奨と設定画面([§11](./11-settings.md))への案内。
- **基本の流れ**: 番号付きリスト(6ステップ)で、マスタ登録([§9](./09-masters.md))→機器登録([§3](./03-equipment-form.md))→項目登録([§6](./06-service-item-modal.md)、機器詳細/点検校正項目一覧から起動)→実施記録登録([§7](./07-service-record-modal.md)、点検校正項目一覧から起動)→案件作成→案件の進捗管理([§8](./08-service-orders.md))の順に、各ステップから遷移可能な画面へのリンクを付す。案件作成のステップでは点検校正項目一覧([§5](./05-service-item-list.md))の行アクション「案件」(外部かつ有効案件なしの項目のみ表示)から案件作成モーダルで案件を作成すること、進捗管理のステップでは作成した案件が点検校正外部案件画面のボードに表示される、というデータの流れを明記する。
- **ステータスの見方**: [§0.3](./README.md#03-ステータスバッジ色共通定義) の5ステータスを定義順(優先度順: overdue→orderNow→inProgress→dueSoon→ok)にバッジ表示し、各ステータスの意味を平易な日本語で説明する。
- **期限と発注推奨日の計算**: ドメインモデル §4.1/§4.2 の計算式をそのまま記載する(次回期限 = 前回実施日 + 周期、月単位の加算は暦月ベース。発注推奨日(外部のみ) = 次回期限 − 納期 − 発注余裕日数、納期は項目個別設定がなければ依頼先の標準納期を使用)。
- **各画面の説明**: ダッシュボード([§1](./01-dashboard.md))・機器一覧/機器詳細([§2](./02-equipment-list.md)/[§4](./04-equipment-detail.md))・点検校正項目一覧([§5](./05-service-item-list.md))・点検校正外部案件([§8](./08-service-orders.md))・メーカー/取引先・担当者([§9](./09-masters.md))・通知([§10](./10-notifications.md))・設定([§11](./11-settings.md)、PWAとしてのインストール、CSVによるデータのエクスポート/インポート、データ全削除)の各画面へのリンクと、各画面の目的を2〜3文で要約する。
- **バックアップと復元**: 設定画面([§11](./11-settings.md))からのCSVエクスポート/インポートの案内と、端末変更・ブラウザデータ消去に備えた定期エクスポートの推奨。CSVの仕様と手順を利用者向けの平易な日本語で詳述する(D-054)。内容は §11 の仕様(D-028〜D-030・D-053)の言い換えであり、矛盾させない:
  - **CSVエクスポート**: データの種類ごとに1ファイル、UTF-8 BOM付き(Excel互換)、ファイル名は種類+出力日(例 `equipment_2025-01-31.csv`)、0件でも列名のみのCSVを出力できること、エクスポートしたCSVはそのまま同種のインポートで復元可能(ラウンドトリップ、D-028)、1行目(英語の列名ヘッダ)は種類の照合に使うため変更しないこと。
  - **CSVインポート(復元)の手順**: 番号付きリスト4ステップ。(1) 設定画面で対象の種類を選択 → (2) ファイル選択で自動チェック、取り込み可能行数とエラー・警告を行番号付きでプレビュー表示 → (3) エラー1件以上で「確定」非活性、エラー行のみ除外する取り込みはない(D-030)ためCSV修正後に選び直し → (4) エラー0件で「確定」→ 確認ダイアログ → 選択した種類の既存データを取り込み内容で全置換(追記・部分更新ではない、D-029)。
  - **インポート時にチェックされる内容**: ヘッダの種類一致・各行の列数・セル形式(必須/日付 YYYY-MM-DD/数値/true・false/選択肢)・ファイル内重複(id、機器は managementNo も)・参照先の存在(現在登録されているデータと突合)を列挙する。あわせて数式として解釈され得る値の警告(D-053)がエラーとは別枠で取り込みを妨げないことを記載する。
  - **複数の種類を復元する順番(D-054)**: 参照チェックが現在ストアとの突合であることから、依存順のインポートを案内する: メーカー/取引先・担当者 → 機器 → 点検校正項目 → 点検校正外部案件 → 実施記録 → 通知(実施記録は案件を、通知は項目/案件・担当者を参照するため後ろ)。
  - あわせてCSVインジェクション対策(D-053)の運用注意を記載する: 出所の分からないCSVファイルをインポートしないこと、エクスポートしたCSVをExcel等で開いた際に数式の実行を確認する警告が出た場合は内容を確認するまで許可しないこと。
- **ライセンスとソースコード**: 本アプリがMITライセンスのオープンソースソフトウェアであることと、ソースコードのGitHubリポジトリ(<https://github.com/KyoheiTashiro/calibration-manager>)への外部リンク(`target="_blank"` + `rel="noreferrer"`)を記載する。あわせて、ログイン機能・アカウントごとの権限管理・DBサーバーによる複数人でのデータ共有などは本アプリのスコープ外であり、ライセンスの条件(GitHubリポジトリの LICENSE.ja.md ファイル(日本語参考訳) <https://github.com/KyoheiTashiro/calibration-manager/blob/main/LICENSE.ja.md> への外部リンク、同じく `target="_blank"` + `rel="noreferrer"`)に従い利用者が自由に改変・拡張できることを案内する。
