# 12. 利用マニュアル

[画面設計 共通仕様・全体構成](./README.md) の一部。ステータス色・モーダル共通挙動・確認ダイアログ等の共通仕様は [README(§0 共通仕様)](./README.md#0-共通仕様) を参照。

**目的**: アプリの目的・基本操作の流れ・ステータスの見方・期限計算式・各画面の説明・CSVエクスポート/インポートの使い方・ライセンスをまとめた利用マニュアルを提供する。

**URL**: `/manual`

**非機能要件**: 静的コンテンツのみで構成し、store(Zustand)を一切参照しない。モーダルなし・フォーム入力なし・確認ダイアログなし。サイドバー最下部(設定の次)に配置する。表示のみのローカルUI状態(目次アコーディオンの開閉(D-057)・インポートチェック表のタブ切替(D-056))は静的コンテンツの範囲内として持ってよい。

## 表示項目 / 機能

- **目次(D-057)**: ページ先頭(h1直下)に `<details>` によるアコーディオン目次を置き、全セクション(ご利用にあたって〜ライセンスとソースコード)へジャンプできる。HashRouter とページ内アンカー(`href="#..."`)は競合するため、リンクではなく button + `scrollIntoView({ behavior: "smooth" })` で実現する。
- **ページ上部へ戻るボタン(D-063)**: 本画面は縦に長いため、画面右下に固定表示(fixed)のフローティングボタンを置き、クリックでページ先頭(h1)へ戻れるようにする。ラベルは上向き矢印アイコン(`ArrowUpIcon`)のみとし、`aria-label="ページ上部へ戻る"` を付与する。スクロールコンテナは window ではなくレイアウトの main 要素のため、目次(D-057)と同じ `scrollIntoView({ behavior: "smooth" })` 方式で実現する。
- **ご利用にあたって**: アプリの目的(点検・校正期限の一元管理)、データが LocalStorage に保存されサーバへ送信されないこと(オフラインでも利用可)、ブラウザ・端末ごとにデータが独立していること、定期バックアップの推奨と設定画面([§11](./11-settings.md))への案内。
- **基本の流れ**: 番号付きリスト(6ステップ)で、マスタ登録([§9](./09-masters.md))→機器登録([§3](./03-equipment-form.md))→項目登録([§6](./06-service-item-modal.md)、機器詳細/点検校正項目一覧から起動)→実施記録登録([§7](./07-service-record-modal.md)、点検校正項目一覧から起動)→案件作成→案件の進捗管理([§8](./08-service-orders.md))の順に、各ステップから遷移可能な画面へのリンクを付す。案件作成のステップでは点検校正項目一覧([§5](./05-service-item-list.md))の行アクション「案件」(外部かつ有効案件なしの項目のみ表示)から案件作成モーダルで案件を作成すること、進捗管理のステップでは作成した案件が点検校正外部案件画面のボードに表示される、というデータの流れを明記する。
- **ステータスの見方**: [§0.3](./README.md#03-ステータスバッジ色共通定義) の5ステータスを定義順(優先度順: overdue→orderNow→inProgress→dueSoon→ok)にバッジ表示し、各ステータスの意味を平易な日本語で説明する。
- **期限と発注推奨日の計算**: ドメインモデル §4.1/§4.2 の計算式をそのまま記載する(次回期限 = 前回実施日 + 周期、月単位の加算は暦月ベース。発注推奨日(外部のみ) = 次回期限 − 納期 − 発注余裕日数、納期は項目個別設定がなければ依頼先の標準納期を使用)。
- **各画面の説明**: ダッシュボード([§1](./01-dashboard.md))・機器一覧/機器詳細([§2](./02-equipment-list.md)/[§4](./04-equipment-detail.md))・点検校正項目一覧([§5](./05-service-item-list.md))・点検校正外部案件([§8](./08-service-orders.md))・メーカー/取引先・担当者([§9](./09-masters.md))・通知([§10](./10-notifications.md))・設定([§11](./11-settings.md)、PWAとしてのインストール、CSVによるデータのエクスポート/インポート、データ全削除)の各画面へのリンクと、各画面の目的を2〜3文で要約する。
- **CSVエクスポートとインポート**: 設定画面([§11](./11-settings.md))からのCSVエクスポート/インポートの案内。導入で「エクスポートはバックアップとして、インポートはバックアップからの復元や新しいデータの一括登録に利用できる」という用途を示し、端末の故障・変更やブラウザデータ消去に備えた定期エクスポートを推奨する。CSVの仕様と手順を利用者向けの平易な日本語で詳述する(D-054)。内容は §11 の仕様(D-028〜D-030・D-053)の言い換えであり、矛盾させない:
  - **CSVエクスポート**: データの種類ごとに1ファイル、UTF-8 BOM付き(Excel互換)、0件でも1行目(項目名)のみのCSVを出力できること、エクスポートしたCSVはそのまま同種のインポートで復元可能(ラウンドトリップ、D-028)、1行目(英語の項目名ヘッダ)は種類の照合に使うため変更しないこと。
  - **エクスポートファイル名の一覧表(D-065)**: 種類ごとのファイル名を「データの種類・ファイル名」の2列の表で示す(列見出しは技術用語「論理名」を避け D-062 の方針に従う)。ファイル名は `vendors_YYYY-MM-DD.csv` のように `YYYY-MM-DD` をプレースホルダーとして表記し、表の直前で出力日が入ることを例(`equipment_2025-01-31.csv`)とともに案内する。行順は推奨インポート順(D-058/D-060 と共通)。表のラベル・ファイル名は `ENTITY_CSV_SPECS` と、設定画面のダウンロード処理と共有するファイル名生成関数 `entityCsvFileName` から導出し、実装との乖離を防ぐ。
  - **CSVインポート**: 番号付きリスト4ステップ。(1) 設定画面で対象の種類を選択 → (2) ファイル選択で自動チェック、取り込み可能行数とエラー・警告を行番号付きでプレビュー表示 → (3) エラー1件以上で「確定」非活性、エラー行のみ除外する取り込みはない(D-030)ためCSV修正後に選び直し → (4) エラー0件で「確定」→ 確認ダイアログ → 選択した種類の既存データを取り込み内容で全置換(追記・部分更新ではない、D-029)。手順の後に続けて、インポートはエクスポート済みCSVの復元専用ではなく、列仕様(D-028)を満たすCSVであれば表計算ソフト等で作成した新しいデータの一括登録にも使えること(D-055)、id 列はファイル内で重複しない任意の文字列でよいこと、既存データに追加する場合は全置換(D-029)のため「同じ種類をエクスポート → 行を追記 → インポート」の手順を踏むことを記載する(独立した小見出しは設けず本小見出し内に含める)。
  - **インポート時にチェックされる内容**: ヘッダの種類一致・各行の項目数(列数)・セル形式(必須/日付 YYYY-MM-DD/数値/true・false/選択肢)・ファイル内重複(id、機器は managementNo も)・参照先の存在(現在登録されているデータと突合)を列挙する。あわせて数式として解釈され得る値の警告(D-053)がエラーとは別枠で取り込みを妨げないことを記載する。
  - **種類別の列仕様表(D-056)**: 共通チェックの列挙に続けて、種類(7種)をタブで切り替えて各列の「項目名・画面での名前・必須・チェック内容」を示す表を表示する。「項目名」はCSVの1行目に書く英語の列名(スキーマのキー)。「画面での名前」はCSV列とアプリ画面上の項目を対応付けるための日本語名で、各画面のフォーム・一覧で使っている表示ラベルと一致させる(利用者が画面で見る名前と同じ語で引けるようにする)(D-061)。マニュアルの利用者向け文言(表見出し・本文)では技術用語の「列」を使わず「項目」と表記する(D-062)。「必須」列は○の有無で表現する(必須は○、任意は空欄、条件付きのみ「条件付き」と表記し条件はチェック内容に記載)。表の直前で凡例(「項目名」はCSVの1行目(見出し行)に記載する英語の列名、「画面での名前」はアプリ画面上の表示名、○は入力必須、空欄は任意)を案内する。表データは `features/manual/importCheck/columns.ts` に定数として持ち、列名と列順が `ENTITY_CSV_SPECS` の shape と一致することをテストで強制する(zodスキーマとの乖離防止)。列挙値の説明は既存の日本語ラベル定数(`features/*/constants.ts`)から `値(ラベル)` 形式で生成し、選択式であることを明示するため末尾に「のいずれか」を付ける。boolean 列も同じ形式にそろえ、列ごとに true/false の意味を示す(例: `true(既読) / false(未読) のいずれか`)(D-059)。タブの並び順は推奨インポート順(D-054 の依存順: メーカー/取引先 → 担当者 → 機器 → 点検校正項目 → 点検校正外部案件 → 実施記録 → 通知)とし、直後に案内するインポート順の記載と一致させる(D-058)。この並びは設定画面([§11](./11-settings.md))のエクスポートボタン・インポート対象セレクトとも共通(D-060)。
  - **複数の種類をインポートする順番(D-054)**: 参照チェックが現在ストアとの突合であることから、依存順のインポートを案内する: メーカー/取引先 → 担当者 → 機器 → 点検校正項目 → 点検校正外部案件 → 実施記録 → 通知(実施記録は案件を、通知は項目/案件・担当者を参照するため後ろ)。
  - **セキュリティ上の注意**: CSVインジェクション対策(D-053)の運用注意を独立した小見出しで記載する: インポートするCSVは自身でエクスポートしたもの・入手元が信頼できるものだけにすること、エクスポートしたCSVをExcel等で開いた際に数式の実行を確認する警告が出た場合は内容に心当たりがあるか確認できるまで「許可しない」を選ぶこと(禁止形ではなく推奨形の文体で案内する)。
- **ライセンスとソースコード**: 本アプリがMITライセンスのオープンソースソフトウェアであることと、ソースコードのGitHubリポジトリ(<https://github.com/KyoheiTashiro/calibration-manager>)への外部リンク(`target="_blank"` + `rel="noreferrer"`)を記載する。あわせて、ログイン機能・アカウントごとの権限管理・DBサーバーによる複数人でのデータ共有などは本アプリのスコープ外であり、ライセンスの条件(GitHubリポジトリの LICENSE.ja.md ファイル(日本語参考訳) <https://github.com/KyoheiTashiro/calibration-manager/blob/main/LICENSE.ja.md> への外部リンク、同じく `target="_blank"` + `rel="noreferrer"`)に従い利用者が自由に改変・拡張できることを案内する。
