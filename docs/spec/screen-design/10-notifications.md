# 10. 通知センター

[画面設計 共通仕様・全体構成](./README.md) の一部。ステータス色・モーダル共通挙動・確認ダイアログ等の共通仕様は [README(§0 共通仕様)](./README.md#0-共通仕様) を参照。

**目的**: アプリ内通知(Notification)の閲覧・既読管理。

**URL**: `/notifications`

## 画面レイアウト

```
┌──────────────────────────────────────────────────────┐
│ 通知センター               [未読(3)][既読]  [全て既読]│
│ ┌──────────────────────────────────────────────────┐│
│ │🔴 [期限超過] EQ-001 年次校正が期限を過ぎています   ││
│ │             2026-07-01                            ││
│ │🟠 [要発注]   EQ-014 校正の発注時期です            ││
│ │             2026-07-02                            ││

│ │             2026-07-03                            ││
│ └──────────────────────────────────────────────────┘│
└──────────────────────────────────────────────────────┘
```

## 表示項目(Notification §3.7)

各行: `type`(種別アイコン+ラベル)、`message`、`createdDate`、既読状態。対象は `targetType`(inspectionItem/order)+ `targetId`。

**通知種別のアイコン/色**:

| type               | ラベル   | 色        | 対象       |
| ------------------ | -------- | --------- | ---------- |
| `dueSoon`          | 期限接近 | 🟡 黄     | inspectionItem       |
| `overdue`          | 期限超過 | 🔴 赤     | inspectionItem       |
| `orderRecommended` | 要発注   | 🟠 橙     | inspectionItem(外部) |
| `deliveryDueSoon`  | 納期接近 | 🟣 紫     | order      |
| `deliveryOverdue`  | 納期超過 | 🔴 赤(濃) | order      |

## 操作・アクション

- タブ: `未読`(件数バッジ)/ `既読`。
- 行クリック → まず `markAsRead`(`isRead=true`)、次に遷移(D-027):
  - `targetType=order` → `/orders`(カードのハイライトは不実装)。
  - `targetType=inspectionItem` → 項目から機器を辿り機器詳細 `/equipment/:id` へ遷移。
  - 対象が dangling(参照先削除済み)の場合は既読化のみで遷移しない。
- 「全て既読」→ 未読すべてを `isRead=true`。

## 通知生成仕様(ドメインモデル §3.7 に準拠)

- **生成タイミング**: (1)マウント時に即スキャン (2)60秒間隔のポーリング (3)`visibilitychange` で可視復帰時、の3経路で今日の日付を前回スキャン日(`useRef`、非永続)と比較し、異なる場合のみ全対象をスキャンして生成する(D-025)。
- **生成条件**: ドメインモデル §3.7 の通知種別テーブルの発生条件に従う(dueSoon/overdue/orderRecommended/deliveryDueSoon/deliveryOverdue)。対象は稼働機器の有効項目、および有効な案件。
- **重複抑止**: 同一対象(targetType+targetId)・同一 type の**未読**通知が既に存在する場合は新規生成しない。
- **宛先**: 対象項目の personId。担当者が無効化済みの場合もフォールバックしない。既存通知・項目の personId は元のまま保持し、無効化後も当該 personId 宛の通知を生成・表示し続ける。UIは必要箇所で担当者名に「(無効)」を注記する(D-001)。

## 表示ルール

- 各タブ内は createdDate 降順。
- ヘッダーのベルバッジ([§0.1](./README.md#01-全体構成))は未読件数と同期。

## 空状態

- 未読タブ0件: 「未読の通知はありません」。既読タブ0件: 「既読の通知はありません」。
