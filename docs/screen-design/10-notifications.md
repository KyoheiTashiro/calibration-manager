# 10. 通知センター

[画面設計 共通仕様・全体構成](./README.md) の一部。ステータス色・モーダル共通挙動・確認ダイアログ等の共通仕様は [README(§0 共通仕様)](./README.md#0-共通仕様) を参照。

**目的**: アプリ内通知(Notification)の閲覧・既読管理。

**URL**: `/notifications`

## 画面レイアウト

```
┌──────────────────────────────────────────────────────┐
│ 通知センター               [未読(3)][既読]  [全て既読]│
│ ┌──────────────────────────────────────────────────┐│
│ │🔴 [期限超過] EQ-001 年次校正が期限を過ぎています   ││
│ │             2026-07-01                            ││
│ │🟠 [要発注]   EQ-014 校正の発注時期です            ││
│ │             2026-07-02                            ││
│ │🟣 [納期接近] ORD-0007 返却予定日が近づいています   ││
│ │             2026-07-03                            ││
│ └──────────────────────────────────────────────────┘│
└──────────────────────────────────────────────────────┘
```

## 表示項目(Notification §3.7)

各行: `type`(種別アイコン+ラベル)、`message`、`createdDate`、既読状態。対象は `targetType`(item/order)+ `targetId`。

**通知種別のアイコン/色**:

| type               | ラベル   | 色        | 対象       |
| ------------------ | -------- | --------- | ---------- |
| `dueSoon`          | 期限接近 | 🟡 黄     | item       |
| `overdue`          | 期限超過 | 🔴 赤     | item       |
| `orderRecommended` | 要発注   | 🟠 橙     | item(外部) |
| `deliveryDueSoon`  | 納期接近 | 🟣 紫     | order      |
| `deliveryOverdue`  | 納期超過 | 🔴 赤(濃) | order      |

## 操作・アクション

- タブ: `未読`(件数バッジ)/ `既読`。
- 行クリック → 対象へ遷移し既読化(`isRead=true`):
  - `targetType=item` → `/items`(該当項目)または該当機器詳細。
  - `targetType=order` → `/orders`(該当カード)。
- 「全て既読」→ 未読すべてを `isRead=true`。

## 通知生成仕様(ドメインモデル §3.7 に準拠)

- **生成タイミング**: アプリ起動時、および日付変更検知時(前回スキャン日と今日が異なる場合)に全対象をスキャンして生成。
- **生成条件**: ドメインモデル §3.7 の通知種別テーブルの発生条件に従う(dueSoon/overdue/orderRecommended/deliveryDueSoon/deliveryOverdue)。対象は稼働機器の有効項目、および有効な案件。
- **重複抑止**: 同一対象(targetType+targetId)・同一 type の**未読**通知が既に存在する場合は新規生成しない。
- **宛先**: 対象項目の personId。※担当者が無効化済みの場合のフォールバックは**未決**(ドメインモデル §8。当面は当該 personId のまま生成)。

## 表示ルール

- 各タブ内は createdDate 降順。
- ヘッダーのベルバッジ([§0.1](./README.md#01-全体構成))は未読件数と同期。

## 空状態

- 未読タブ0件: 「未読の通知はありません」。既読タブ0件: 「既読の通知はありません」。
