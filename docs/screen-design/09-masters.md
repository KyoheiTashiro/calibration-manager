# 9. マスタ管理(メーカー/取引先・担当者)

[画面設計 共通仕様・全体構成](./README.md) の一部。ステータス色・モーダル共通挙動・確認ダイアログ等の共通仕様は [README(§0 共通仕様)](./README.md#0-共通仕様) を参照。

`/vendors` と `/persons` の2画面。いずれもテーブル + 追加/編集モーダル構成。

## 9-A. メーカー/取引先マスタ

**目的**: Vendor(メーカー兼校正業者)の登録・編集。

**URL**: `/vendors`

```
┌──────────────────────────────────────────────────────┐
│ メーカー/取引先                          [+ 追加]     │
│ 名称        メーカー 校正業者 標準納期 窓口   連絡先  │
│ ミツトヨ      ✓       ✓      30日   山田  03-...   │
│ エー・アンド  ✓       ✓      21日   —    —        │
│ 日本測器      —       ✓      45日   佐々木 04-...  │
└──────────────────────────────────────────────────────┘
```

**追加/編集モーダル(Vendor §3.1)**:

| フィールド   | 属性                 | 必須 | 形式                                   |
| ------------ | -------------------- | ---- | -------------------------------------- |
| 名称         | name                 | ○    | テキスト                               |
| メーカー     | isManufacturer       | ○    | チェックボックス                       |
| 校正業者     | isCalibrator         | ○    | チェックボックス                       |
| 窓口担当者   | contactPerson        |      | テキスト                               |
| メール       | email                |      | email形式                              |
| 電話         | phone                |      | テキスト                               |
| 標準納期(日) | standardLeadTimeDays |      | 数値。**isCalibrator=true 時のみ表示** |
| 備考         | note                 |      | テキスト                               |

- バリデーション: `name` 必須。`isManufacturer`/`isCalibrator` は真偽値(両方false も許容するが警告表示: どちらの用途にも出てこない)。`email` は入力時に email 形式。`standardLeadTimeDays` は0以上。
- **削除ガード**: 参照されている Vendor(Equipment.manufacturerId / InspectionItem.vendorId / CalibrationOrder.vendorId のいずれかから参照)は削除不可。削除ボタン押下時に「この取引先は参照されているため削除できません」を表示。未参照時のみ確認ダイアログ後に削除。
- 空状態: 「取引先が未登録です」+「+ 追加」。

## 9-B. 担当者マスタ

**目的**: Person の登録・編集・無効化。

**URL**: `/persons`

```
┌──────────────────────────────────────────────┐
│ 担当者                            [+ 追加]    │
│ 氏名   部署     メール           状態         │
│ 田中   品質保証 tanaka@ex.jp     有効 [編集]  │
│ 鈴木   製造     suzuki@ex.jp     有効 [編集]  │
│ 佐藤   品質保証 sato@ex.jp       無効 [編集]  │
└──────────────────────────────────────────────┘
```

**追加/編集モーダル(Person §3.2)**:

| フィールド | 属性       | 必須 | 形式                     |
| ---------- | ---------- | ---- | ------------------------ |
| 氏名       | name       | ○    | テキスト                 |
| メール     | email      | ○    | email形式                |
| 部署       | department |      | テキスト                 |
| 有効       | isActive   | ○    | チェックボックス(トグル) |

- バリデーション: `name` 必須、`email` 必須・email 形式。
- **削除の代わりに無効化**: 物理削除は行わず `isActive=false` にする(ドメインモデル §3.2)。無効化は確認ダイアログ付き。
- **無効化警告**: 無効化しようとする Person が有効な InspectionItem(isActive=true)に割り当たっている場合、「この担当者は現役の点検校正項目 N 件に割り当てられています。通知が届かなくなる可能性があります」と警告表示。※通知の宛先フォールバック先は**未決**(ドメインモデル §8)。
- 空状態: 「担当者が未登録です」+「+ 追加」。
